## Template Customization Guide

### 1. Replace Placeholders

**[skill-name]**: Kebab-case name (e.g., `docker-compose-manager`)

**[file format/command structure]**: What you're documenting (e.g., "Docker Compose files", "GitHub Actions workflows")

**[key syntax elements]**: Specific technical details (e.g., "service definitions, volumes, networks")

**[action 1-4]**: Specific user actions that trigger this skill (e.g., "writing docker-compose.yml files", "defining multi-container applications")

### 2. Adapt Structure

**For simple CLI skills** (150-300 lines):
- Remove @REFERENCE.md, @SYNTAX.md, @EXAMPLES.md references
- Integrate all content into main SKILL.md
- Keep templates inline
- Simplify sections

**For moderate CLI skills** (300-600 lines):
- Keep main SKILL.md structure
- Create @EXAMPLES.md for extensive examples
- Use templates/ directory if 3+ templates
- Reference external files as needed

**For comprehensive CLI skills** (600+ lines):
- Use all sections
- Create separate @SYNTAX.md, @EXAMPLES.md, @TIPS.md, @REFERENCE.md
- Extensive templates/ directory
- Main SKILL.md becomes orchestrator with @ references

### 3. Prioritize Key Sections

**Must have:**
- Complete syntax reference
- Validation checklist (15-30 items)
- Basic template (minimal working)
- Troubleshooting (top 5-10 issues)
- Configuration options with trade-offs

**Should have:**
- Intermediate and advanced templates
- Extensive examples
- Best practices section
- Edge cases documentation
- Creation process workflow

**Nice to have:**
- Separate reference files
- Advanced configuration patterns
- Integration guidance
- Team collaboration notes

### 4. Format-Specific Adjustments

**YAML-based formats:**
- Emphasize indentation rules (spaces not tabs)
- Document quote handling
- Show multi-line syntax
- Explain anchor/alias if applicable

**JSON-based formats:**
- Emphasize comma rules
- Document schema validation
- Show nesting patterns
- Explain schema references

**Custom DSL/syntax:**
- Character-by-character breakdown
- BNF or EBNF grammar if complex
- Tokenization rules
- Parser behavior

**CLI commands:**
- Flag and option documentation
- Argument ordering rules
- Input/output format
- Exit codes and errors

### 5. Testing Your CLI Skill

Before finalizing:
1. [ ] Every syntax example works when copy-pasted
2. [ ] All validation commands execute successfully
3. [ ] Templates are tested and functional
4. [ ] Troubleshooting solutions actually fix the issues
5. [ ] Configuration examples demonstrate trade-offs
6. [ ] Description triggers skill when expected
7. [ ] @ references point to existing files
8. [ ] Checklist items are specific and verifiable
