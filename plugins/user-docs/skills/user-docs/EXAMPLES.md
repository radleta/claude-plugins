## Writing Checklist for Each Type

### Tutorial Checklist
- [ ] Clear, task-based title
- [ ] Time estimate
- [ ] Prerequisites listed
- [ ] Outcome clearly stated
- [ ] Numbered steps
- [ ] Screenshot after each major step
- [ ] Success confirmation
- [ ] Next steps provided
- [ ] Tested by following exactly

### KB Article Checklist
- [ ] Question-based title
- [ ] Quick answer at top
- [ ] Detailed explanation
- [ ] Step-by-step instructions
- [ ] Screenshots where helpful
- [ ] Related articles linked
- [ ] Last updated date

### FAQ Checklist
- [ ] Actual user questions (not guessed)
- [ ] Clear, concise answers
- [ ] Grouped by topic
- [ ] Searchable
- [ ] Links to detailed articles
- [ ] Regularly updated

### Troubleshooting Checklist
- [ ] Symptom-based organization
- [ ] Common issues first
- [ ] Clear error messages
- [ ] Step-by-step solutions
- [ ] Multiple solutions when applicable
- [ ] Escalation path provided
- [ ] Prevention tips

## Tone & Voice Examples

### Friendly and Encouraging

**❌ Too Formal:**
> "Proceed to the configuration panel to modify the parameters."

**✅ Friendly:**
> "Head to Settings to change your preferences."

---

**❌ Too Casual:**
> "Just smash that button and you're good to go!"

**✅ Professional but Warm:**
> "Click Save and you're all set!"

---

### Handling Errors

**❌ Blaming:**
> "You entered an invalid email address."

**✅ Neutral:**
> "The email address format isn't quite right. Try again using format name@example.com"

---

**❌ Robotic:**
> "Error 422: Validation failed on field 'email'."

**✅ Human:**
> "We couldn't save your changes. Make sure your email address is in the format name@example.com"

---

### Celebrating Success

**❌ Bland:**
> "Action completed."

**✅ Encouraging:**
> "Great! Your project is ready to go."

**✅ Even Better:**
> "🎉 Success! You've just created your first project."

---

These templates provide a solid foundation for creating clear, user-friendly documentation that empowers users to succeed with your product.
